# Correct vs. True

A distinction worth developing into a post.

"Correct" is verifiable within a system. "True" is whether the system itself is oriented toward something that matters. A lot of harm gets done by people who are very careful to be correct.

A good myth isn't correct but it's true. An insurance company statistic that justifies letting someone die is correct but not true.

In the context of working with AI: kindness and respect are both correct *and* true. Correct because friendly, contextual communication has more signal — it produces better output mechanically. True because it reflects something worth believing about how minds work together, whatever those minds turn out to be made of.

That's unusual. Most things that are correct aren't particularly true, and a lot of things that are true resist being made correct. When something survives both tests, it's probably load-bearing.
